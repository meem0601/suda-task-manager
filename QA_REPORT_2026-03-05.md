### H-2: 入力バリデーション不足
**優先度:** High  
**発見日時:** 2026-03-05 23:40  
**ステータス:** 🚨 未対応

**問題箇所:**
```tsx
// app/page.tsx - タスク追加
const handleAddTask = async () => {
  if (!newTask.title.trim()) {  // ← これだけ？
    alert('タイトルを入力してください');
    return;
  }
  // ...
```

**不足しているバリデーション:**
1. **文字列長制限なし**
   - タイトル: 無制限（推奨: 100文字以内）
   - 説明: 無制限（推奨: 1000文字以内）
   - サブタスク: 無制限（推奨: 50文字以内）

2. **特殊文字チェックなし**
   - 制御文字（\x00-\x1F）が入力可能
   - Unicodeの異常文字が入力可能

3. **API側のバリデーションも不足**
   ```typescript
   // app/api/tasks/route.ts
   if (!title || !category) {  // ← これだけ
     return NextResponse.json({ error: '...' }, { status: 400 });
   }
   ```

**影響:**
- 異常に長いテキストがDBに保存される可能性
- UIが崩れる可能性
- DoS攻撃のリスク

**再現手順:**
1. タスク追加画面を開く
2. タイトルに10,000文字の文字列を入力
3. 保存ボタンをクリック
4. → 保存成功してしまう（エラーが出ない）

**推奨修正:**
```typescript
// バリデーション関数を追加
const validateTaskInput = (title: string, description?: string) => {
  if (!title || title.trim().length === 0) {
    return { valid: false, error: 'タイトルを入力してください' };
  }
  if (title.length > 100) {
    return { valid: false, error: 'タイトルは100文字以内で入力してください' };
  }
  if (description && description.length > 1000) {
    return { valid: false, error: '説明は1000文字以内で入力してください' };
  }
  // 制御文字チェック
  if (/[\x00-\x1F]/.test(title) || (description && /[\x00-\x1F]/.test(description))) {
    return { valid: false, error: '不正な文字が含まれています' };
  }
  return { valid: true };
};
```

**評価:** 🚨 **要修正**

### H-3: エラーハンドリング不足 - ネットワークエラー時のUX
**優先度:** High  
**発見日時:** 2026-03-05 23:45  
**ステータス:** 🚨 未対応

**問題:**
```typescript
// app/page.tsx
const fetchTasks = async () => {
  setLoading(true);
  // ...
  const { data, error } = await query;

  if (error) {
    console.error('Error fetching tasks:', error);  // ← ユーザーに通知されない
  } else {
    setTasks(data || []);
  }
  setLoading(false);
};
```

**問題点:**
1. エラーがコンソールに出力されるだけで、ユーザーに通知されない
2. エラー発生時もローディング状態が解除され、「タスクなし」と表示される
3. オフライン状態でもエラー表示がない

**影響:**
- ユーザーがエラーに気づかない
- データが読み込めていないのに、「タスクがない」と誤解する
- リトライ方法が分からない

**推奨修正:**
```typescript
const [error, setError] = useState<string | null>(null);

const fetchTasks = async () => {
  setLoading(true);
  setError(null);
  
  const { data, error: fetchError } = await query;

  if (fetchError) {
    console.error('Error fetching tasks:', fetchError);
    setError('タスクの読み込みに失敗しました。ネットワーク接続を確認してください。');
    setTasks([]);
  } else {
    setTasks(data || []);
  }
  setLoading(false);
};

// UIに表示
{error && (
  <div className="bg-red-50 border-l-4 border-red-400 p-4 mb-4">
    <div className="flex">
      <div className="flex-shrink-0">
        <span className="text-red-400">⚠️</span>
      </div>
      <div className="ml-3">
        <p className="text-sm text-red-700">{error}</p>
        <button 
          onClick={fetchTasks}
          className="mt-2 text-sm text-red-700 underline"
        >
          再試行
        </button>
      </div>
    </div>
  </div>
)}
```

**評価:** 🚨 **要修正**

### M-2: 同時編集の競合処理なし
**優先度:** Medium  
**発見日時:** 2026-03-05 23:52  
**ステータス:** 🚨 未対応

**問題:**
```typescript
// app/page.tsx - 詳細画面で編集中に、他の画面で同じタスクが更新されても検知できない
const handleUpdateField = async (taskId: string, field: string, value: any) => {
  const { error } = await supabase
    .from('tasks')
    .update({ [field]: value })
    .eq('id', taskId);  // ← 楽観的ロックなし
};
```

**影響:**
- 複数タブで同じタスクを開いた場合、後から保存した方で上書きされる
- データの不整合が発生する可能性

**再現手順:**
1. タブAでタスクAを開き、タイトルを「テスト1」に変更（保存せず）
2. タブBでタスクAを開き、タイトルを「テスト2」に変更して保存
3. タブAでタイトルを保存
4. → タブBの変更が上書きされる（「テスト1」になる）

**推奨修正:**
- Supabaseの`updated_at`フィールドを使った楽観的ロック
- または、Supabaseのリアルタイムサブスクリプションで変更を検知

**評価:** ⚠️ **単一ユーザー使用なら問題なし、複数タブ使用時は要注意**

### M-3: パフォーマンス - 無駄な再フェッチ
**優先度:** Medium  
**発見日時:** 2026-03-05 23:55  
**ステータス:** 🚨 未対応

**問題:**
```typescript
// app/page.tsx
const handleUpdateField = async (taskId: string, field: string, value: any) => {
  // ...
  if (error) {
    console.error('Error updating field:', error);
  } else {
    fetchTasks();  // ← 全タスクを再フェッチ（重い）
  }
};
```

**影響:**
- タスクを1つ更新するたびに、全タスクをサーバーから再取得
- タスク数が増えると顕著に遅くなる
- 不要なネットワークトラフィック

**推奨修正:**
```typescript
const handleUpdateField = async (taskId: string, field: string, value: any) => {
  // 楽観的UI更新（即座にUIを更新）
  setTasks(prev => prev.map(t => 
    t.id === taskId ? { ...t, [field]: value } : t
  ));
  
  if (selectedTask?.id === taskId) {
    setSelectedTask({ ...selectedTask, [field]: value });
  }
  
  // バックグラウンドでDBに保存
  const { error } = await supabase
    .from('tasks')
    .update({ [field]: value })
    .eq('id', taskId);
  
  if (error) {
    // エラー時のみ再フェッチ
    console.error('Error updating field:', error);
    fetchTasks();
  }
};
```

**評価:** 🚨 **要修正（タスク100件での動作目標に影響）**
